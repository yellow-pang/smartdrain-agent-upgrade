# 17 배포 운영 런북

> 문서 상태: 현재 설정 기준. 실제 운영 인프라 접근·성공 기록은 별도 확인이 필요하다.

## 1. 구조와 준비

운영 Compose는 Nginx만 외부 공개하고 `/`·`/api`·`/ws`를 각각 frontend·backend에 proxy한다. `/docs`, `/openapi.json`, `/redoc`은 운영 Nginx에서 404다. 개발 Compose override는 8080 포트와 FastAPI 문서 proxy를 제공한다.

Jenkinsfile의 현재 설정은 `DEPLOY_DIR=/home/yp/apps/smart-drain`, custom workspace도 같은 경로다. 이는 **현재 설정**이며 일반 표준 경로가 아니다. 권장 설정은 `DEPLOY_DIR=/home/<user>/apps/smart-drain`처럼 계정별 변수화한 경로다.

배포 전 `.env`(Jenkins Credential Secret File), YOLO host artifact, Docker/Compose, 사용 포트, 디스크 여유 공간을 확인한다. `.env`는 Jenkins가 `DEPLOY_DIR/.env`에 복사한다.

## 2. 실행 순서

1. `docker compose config --quiet`로 설정을 확인한다.
2. 모델 파일 경로와 읽기 권한을 확인한다.
3. Compose build·기동 시 `migrate`가 Alembic head를 적용한 뒤 backend를 기동하는지 확인한다.
4. 필요할 때만 `docker compose --profile seed run --rm seed`를 실행한다.
5. `docker compose ps`, Nginx `/`, backend health, REST, WebSocket을 smoke test한다.
6. 실패 시 `docker compose logs nginx backend ai-service migrate`를 먼저 수집한다.

## 3. 장애 대응 순서

| 증상 | 우선 확인 |
| --- | --- |
| Jenkins workspace/배포 경로 불일치 | Jenkinsfile의 `customWorkspace`, `DEPLOY_DIR`, checkout 위치 |
| `.env` 또는 DB 비밀번호 오류 | Secret File 복사 결과와 `COMPOSE_DATABASE_URL` |
| Nginx mount·port 오류 | `nginx/default.conf`, host port 점유, 컨테이너 logs |
| migration 실패 | migrate logs·DB health·Alembic revision |
| 모델 mount 실패 | `SMARTDRAIN_YOLO_MODEL_PATH`, 파일 크기·권한 |
| AI health만 성공 | 실제 샘플 분석과 callback을 별도 확인 |
| 이전 image/container 잔존 | `docker compose ps`와 image 목록 확인 후 계획된 정리 |

`docker compose down -v`, image prune, volume 정리는 데이터를 제거할 수 있으므로 운영에서는 별도 승인과 백업 뒤에만 수행한다. Cloudflare Tunnel 구성은 저장소 설정으로 확인되지 않아 확인 필요다.
