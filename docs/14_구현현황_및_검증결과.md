## 1. 목적과 판단 기준

이 문서는 SmartDrain의 현재 코드를 기준으로 구현 현황과 검증 범위를 구분한다. 코드·migration·배포 설정을 문서 기록보다 우선하며, Plan·Step·PR은 작성 당시의 이력으로 취급한다. 실행하지 않은 검증은 통과로 간주하지 않는다.

## 3. 현재 계약 요약

- 대표 분석 시작 API는 `POST /api/analysis/async-run`이다. `POST /api/analysis/yolo`, `POST /api/analysis/xgboost`는 결과 직접 저장용 경로로 남아 있다.
- AI Service는 `POST /ai/analysis/run`을 BackgroundTasks로 수락한 뒤, `/api/ai-callback/yolo-result`, `/api/ai-callback/xgboost-result`에 순서대로 callback한다. 영속 queue와 failure callback은 구현 확인되지 않았다.
- WebSocket endpoint는 `/ws/drains/status`이며 이벤트는 `DRAIN_STATUS_UPDATED`, `YOLO_RESULT_UPDATED`, `XGBOOST_RESULT_UPDATED`다.
- 실제 센서 DB 필드는 `water_level_cm`, `flow_velocity_mps`다. `quality_status`와 `image_key`는 현재 migration에 없다.
- `risk_score`는 XGBoost 결과의 점수 값이며, 위험 심각도의 보편적·물리적 척도로 해석하면 안 된다.

## 5. 이력·중복 문서 처리

`frontend/docs/plans`, `steps`, `pr`는 작업 이력이며 본문을 현재 구현으로 다시 쓰지 않는다. 현재 기준은 이 문서와 `docs/15`~`docs/17`이다. API 계약의 canonical 후보는 `docs/11_API명세서.md`이며, `frontend/docs/api-spec/*`는 작성 시점 계약 기록으로 유지한다.
