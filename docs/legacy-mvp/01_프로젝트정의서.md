# AI 기반 빗물받이 막힘 탐지 및 침수 위험 모니터링 시스템

## 1. 프로젝트 개요

본 프로젝트는 **침수 위험 지역 내 개별 빗물받이**를 대상으로 막힘 상태와 배수 이상 가능성을 분석하는 AI 기반 도시 침수 모니터링 서비스이다.

MVP에서는 실제 CCTV 및 실제 IoT 센서를 직접 연동하지 않고, **CCTV 스냅샷 이미지 또는 샘플 이미지**와 **수위·유속 모의 센서 데이터**를 활용한다. 이미지는 OpenCV로 전처리하고, YOLO로 막힘 정도와 신뢰도를 산출한다. 이후 YOLO 분석 결과와 센서 데이터를 XGBoost에 입력하여 최종 위험도를 `양호 / 주의 / 위험 / 판단불가`로 분류한다.

관리자는 Kakao Maps API 기반 대시보드에서 빗물받이 위치와 위험 상태를 확인하고, 상세 화면에서 CCTV 이미지, 센서 데이터, YOLO 분석 결과, XGBoost 판단 결과, 위험도 이력을 확인할 수 있다.

---

## 2. 프로젝트 기획 배경

도시 침수는 강우량뿐 아니라 배수 시설의 상태와도 밀접하게 연결된다. 도로 위 빗물을 배수관으로 유입시키는 빗물받이가 낙엽, 담배꽁초, 플라스틱, 공사 잔여물 등으로 막히면 배수 효율이 낮아지고 침수 위험이 커질 수 있다.

기존 침수 관제는 넓은 지역 단위의 수위나 강우량 중심으로 이루어지는 경우가 많다. 그러나 실제 현장 대응에서는 어느 위치의 빗물받이가 막혔는지, 막힘 정도가 어느 수준인지, 수위와 유속이 어떻게 변하고 있는지를 빠르게 확인하는 것이 중요하다.

모든 빗물받이에 고정형 센서를 설치하는 방식은 비용과 유지관리 부담이 크다. 따라서 본 프로젝트는 이미 확보 가능한 CCTV 이미지와 일부 센서 데이터를 함께 활용하여 관리가 필요한 빗물받이를 우선적으로 식별하는 방향으로 기획하였다.

## 3. 프로젝트 목표

본 프로젝트의 목표는 침수 위험 지역 내 빗물받이를 개별 단위로 분석하여 관리자가 위험 상태를 빠르게 확인할 수 있는 MVP를 구현하는 것이다.

| 목표 | 설명 |
|---|---|
| 개별 빗물받이 상태 분석 | CCTV 스냅샷 이미지 또는 샘플 이미지를 기반으로 빗물받이의 막힘 상태를 분석한다. |
| 이미지와 센서 데이터 결합 | YOLO 분석 결과와 수위·유속 모의 센서 데이터를 결합해 이미지 단독 판단의 한계를 보완한다. |
| XGBoost 기반 위험도 판단 | 막힘 비율, 신뢰도, 수위, 유속을 입력값으로 사용해 최종 위험도를 분류한다. |
| 지도 기반 관제 화면 제공 | Kakao Maps API 기반 대시보드에서 빗물받이 위치와 위험 상태를 표시한다. |
| 상태 변화 추적 | 분석 결과와 센서 데이터를 PostgreSQL에 저장하여 위험도 이력을 확인할 수 있도록 한다. |
| 실시간 상태 반영 | WebSocket을 통해 위험도 변경 이벤트를 프론트엔드에 전달한다. |

## 5. 주요 기능

### 5.1 CCTV 스냅샷 이미지 처리

MVP에서는 실제 CCTV 스트림이 아니라 스냅샷 이미지 URL 또는 저장된 샘플 이미지를 활용한다. 백엔드는 이미지를 불러온 뒤 OpenCV로 전처리하고 YOLO 분석 입력으로 전달한다.

### 5.2 OpenCV 기반 이미지 전처리

이미지 분석 품질을 높이기 위해 크기 조정, 밝기 보정, 노이즈 제거, 관심 영역 추출을 수행한다. 전처리 결과는 YOLO 모델의 입력으로 사용된다.

### 5.3 YOLO 기반 빗물받이 상태 분석

YOLO는 이미지에서 빗물받이와 주변 이물질 상태를 분석한다. 주요 출력값은 다음과 같다.

| 출력값 | 설명 |
|---|---|
| `obstruction_ratio` | 막힘 비율, 0~1 범위 |
| `confidence_score` | YOLO 분석 신뢰도, 0~1 범위 |
| `yolo_status` | 이미지 기반 상태 코드 |

### 5.4 XGBoost 기반 최종 위험도 판단

XGBoost는 YOLO 분석 결과와 수위·유속 모의 센서 데이터를 함께 입력받아 최종 위험도를 판단한다.

| 입력값 | 설명 |
|---|---|
| `obstruction_ratio` | 이미지 기반 막힘 정도 |
| `confidence_score` | 이미지 분석 신뢰도 |
| `water_level_cm` | 수위 값 |
| `flow_velocity_mps` | 유속 값 |

| 출력값 | 설명 |
|---|---|
| `risk_score` | 위험 점수 |
| `risk_level` | `good / caution / danger / unknown` 중 하나 |
| `final_decision` | 관리자 화면에 표시할 최종 판단 코드 |

### 5.5 관리자 대시보드

대시보드는 빗물받이 위치와 위험도를 지도 마커로 표시한다. 위험 시설 목록과 상세 패널을 함께 제공하여 관리자가 우선 확인할 대상을 빠르게 찾을 수 있도록 한다.

### 5.6 상세 정보 화면

상세 화면에서는 특정 빗물받이의 최근 이미지, 수위, 유속, 막힘 비율, confidence score, XGBoost 위험도, 위험도 변화 이력을 확인한다.

### 5.7 WebSocket 기반 실시간 상태 갱신

위험도 변경이 발생하면 FastAPI가 WebSocket 이벤트를 전달하고, 프론트엔드는 지도 마커, 위험 시설 목록, 상세 화면의 상태를 즉시 갱신한다.

## 6. 위험도 체계

| 화면 표시 | 내부 코드값 | 의미 |
|---|---|---|
| 양호 | `good` | 현재 침수 위험이 낮고 배수 상태가 안정적인 상태 |
| 주의 | `caution` | 일부 막힘, 수위 상승, 유속 저하 등 모니터링이 필요한 상태 |
| 위험 | `danger` | 침수 가능성이 높아 즉시 확인이 필요한 상태 |
| 판단불가 | `unknown` | 이미지 품질 저하, 센서 누락, 모델 신뢰도 부족 등으로 판단이 어려운 상태 |

## 8. 정리

본 프로젝트는 CCTV 이미지와 센서 데이터를 결합하여 침수 위험 지역 내 개별 빗물받이의 위험도를 판단하는 시스템이다. MVP에서는 샘플 이미지와 모의 센서 데이터를 활용하되, OpenCV, YOLO, XGBoost, PostgreSQL, WebSocket, 지도 대시보드까지 핵심 흐름을 구현한다.

실제 CCTV RTSP 연동, 실제 IoT 센서 MQTT 연동, 대응 요청 프로세스, 외부 알림, LLM 요약은 고도화 기능으로 분리한다.
