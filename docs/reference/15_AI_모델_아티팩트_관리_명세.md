# 15 AI 모델 아티팩트 관리 명세

> 문서 상태: 현재 기준 문서 · 확인 기준일: 2026-06-24

## 1. 구성과 저장 정책

| 구성 | 파일·경로 | Git 포함 | 배포 방식 |
| --- | --- | --- | --- |
| XGBoost | `ai_service/model/sewer_xgboost_model.json` | 포함 | AI Service 이미지에 포함 |
| YOLO | `best.pt` | 확인 필요(현재 작업 트리에 파일 미확인) | host 파일을 `/app/ai_service/model/best.pt`에 읽기 전용 mount |

Compose는 `SMARTDRAIN_YOLO_MODEL_PATH`(기본 `./ai_service/model/best.pt`)를 사용한다. 파일이 없거나 비어 있으면 ai-service는 시작을 중단한다. Jenkins는 Secret File로 `.env`를 `DEPLOY_DIR/.env`에 복사하므로, 모델 파일 경로도 해당 배포 host에서 유효해야 한다.

## 2. 확인 필요 메타데이터

YOLO 모델명·버전, 학습 날짜, 클래스 ID 의미, 입력 이미지 규격, 파일 크기, SHA256, 학습 데이터·평가 결과 위치는 artifact 또는 학습 저장소 없이는 확정할 수 없다. 배포 전 다음을 기록한다.

```bash
sha256sum <best.pt>
```

XGBoost 입력은 코드상 `obstruction_ratio`, `confidence_score`, `water_level_cm`, `flow_velocity_mps` 4개다. `risk_score`의 정확한 확률 보정·성능 수치는 모델 평가 자료를 확인하기 전 발표하지 않는다.

## 3. 교체·롤백 절차

1. 후보 파일의 SHA256, 학습 데이터 버전, 평가 결과를 배포 기록에 남긴다.
2. host의 모델 파일을 원자적으로 교체하고 이전 artifact를 보관한다.
3. Compose 설정과 AI unit/contract test, 샘플 분석을 확인한다.
4. 실패 시 `.env`의 경로를 이전 artifact로 되돌린 뒤 ai-service를 재기동한다.

## 4. 배포 전 확인

- 모델 파일이 일반 파일이고 0 byte가 아닌지 확인한다.
- host 경로, mount 경로, container 파일 권한을 확인한다.
- 모델 변경과 XGBoost feature·callback 계약 변경을 한 배포에 섞지 않는다.
- health endpoint만 통과해도 모델 추론 성공을 의미하지 않으므로 샘플 분석을 별도로 확인한다.
