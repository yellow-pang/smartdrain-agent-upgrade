# 페르소나 및 스트레스 테스트 시나리오

## 1. 문서 개요

본 문서는 `지능형 도시 침수 관리 및 모니터링 시스템`의 주요 사용자와 테스트 시나리오를 정의한다. MVP는 침수 위험 지역 내 개별 빗물받이를 대상으로 CCTV 스냅샷 이미지, YOLO 분석 결과, 수위·유속 모의 센서 데이터, XGBoost 판단 결과를 대시보드에 표시하는 것을 목표로 한다.

시나리오는 단순 성공 케이스보다 오탐, 누락, 센서 이상치, 이미지 품질 저하 등 실제 운영에서 발생할 수 있는 예외 상황을 중심으로 구성한다.

---

## 2. 위험도 체계

| 화면 표시 | 내부 코드값 | 의미 |
|---|---|---|
| 양호 | `good` | 현재 침수 위험이 낮고 배수 상태가 안정적인 상태 |
| 주의 | `caution` | 일부 막힘, 수위 상승, 유속 저하 등 모니터링이 필요한 상태 |
| 위험 | `danger` | 침수 가능성이 높아 즉시 확인이 필요한 상태 |
| 판단불가 | `unknown` | 이미지 품질 저하, 센서 누락, 모델 신뢰도 부족 등으로 판단이 어려운 상태 |

## 3. 주요 페르소나

### 3.1 지자체 관리자

| 항목 | 내용 |
|---|---|
| 주요 목표 | 침수 위험 지역의 빗물받이 상태를 빠르게 확인하고 우선 점검 대상을 선별한다. |
| 주요 사용 화면 | 지도 대시보드, 위험 시설 목록, 상세 화면 |
| 확인 데이터 | 위험도, 위치, 막힘 비율, 수위, 유속, 최근 업데이트 시간 |
| 의사결정 | 위험도가 높은 빗물받이를 우선 확인하고 고도화 단계에서 대응 요청으로 연결한다. |

### 3.2 배수 시설 관리자

| 항목 | 내용 |
|---|---|
| 주요 목표 | 특정 지역의 빗물받이 상태 변화와 반복 위험 여부를 확인한다. |
| 주요 사용 화면 | 상세 화면, 위험도 이력 차트 |
| 확인 데이터 | YOLO 분석 결과, XGBoost 위험도 점수, 센서 이력 |
| 의사결정 | 위험 징후가 반복되는 지점을 유지보수 우선순위에 반영한다. |

### 3.3 시스템 운영자

| 항목 | 내용 |
|---|---|
| 주요 목표 | 데이터 수집, 분석 결과 저장, WebSocket 상태 갱신 흐름을 점검한다. |
| 주요 사용 화면 | 대시보드, 시스템 로그 화면은 고도화 범위 |
| 확인 데이터 | API 응답, DB 저장 상태, 분석 실패 여부, 데이터 누락 여부 |
| 의사결정 | 데이터 누락 또는 분석 실패 발생 시 재수집과 오류 점검을 수행한다. |

## 4. 스트레스 테스트 시나리오

### 시나리오 1. 이미지 인식이 어려운 침수 상황

| 항목 | 내용 |
|---|---|
| 발생 지역 | 고양시 저지대 이면도로 |
| 현장 상황 | 폭우로 도로가 흙탕물에 잠기고 차량 불빛 반사로 이미지 인식 품질이 낮아진다. |
| 입력 데이터 | YOLO 막힘 신뢰도 0.10, 유속 0.0 m/s, 수위 95% |
| 기대 판정 | 위험 |
| 테스트 목적 | 이미지 신뢰도가 낮더라도 수위와 유속 데이터로 침수 위험을 판단하는지 검증한다. |

### 시나리오 2. 수위 센서 이상치

| 항목 | 내용 |
|---|---|
| 발생 지역 | 광진구 상가 밀집 구역 |
| 현장 상황 | 날씨는 맑지만 센서 표면 이물질로 수위 값이 높게 입력된다. |
| 입력 데이터 | YOLO 막힘 신뢰도 0.05, 유속 0.8 m/s, 수위 88% |
| 기대 판정 | 주의 |
| 테스트 목적 | 단일 센서 이상치만으로 즉시 위험으로 분류하지 않고 보수적으로 판단하는지 검증한다. |

| 입력 데이터 | YOLO 막힘 신뢰도 0.65, 유속 0.2 m/s, 수위 40% |
| 기대 판정 | 주의 |
| 테스트 목적 | 침수 발생 전 단계의 예방적 점검 필요성을 감지하는지 검증한다. |

| 테스트 목적 | 이미지와 센서가 모두 높은 위험을 가리킬 때 즉시 위험 상태로 분류하는지 검증한다. |

## 5. 시나리오 활용 기준

| 활용 목적 | 설명 |
|---|---|
| 모델 검증 | YOLO 결과와 센서 데이터가 XGBoost 판단에 함께 반영되는지 확인한다. |
| 화면 시연 | 대시보드에서 양호, 주의, 위험, 판단불가 상태가 구분되어 표시되는지 확인한다. |
| 예외 처리 검증 | 이미지 품질 저하, 센서 누락, 센서 이상치 상황을 테스트한다. |
| 발표 자료 | 서비스가 단순 화면 구현이 아니라 실제 관제 예외 상황을 고려했다는 점을 설명한다. |

---

## 6. 정리

본 시나리오 문서는 MVP의 위험도 판단과 화면 표시를 검증하기 위한 기준이다. 테스트는 단일 데이터가 아니라 이미지 분석 결과, 수위, 유속, 데이터 품질을 함께 고려하여 XGBoost 기반 최종 판단이 적절히 변하는지 확인하는 방향으로 수행한다.
