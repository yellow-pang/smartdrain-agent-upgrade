## PR 제목

[feat] # 대시보드 UI 1차 고도화 완료

## 작업 내용

### 반응형·가독성·렌더링 개선

- 메인 대시보드를 모바일·태블릿·데스크톱 화면에 맞게 재배치했습니다.
  - 모바일/태블릿은 고정 바텀시트 대신 인라인 요약 카드를 사용합니다.
  - 데스크톱은 지도·위험 목록·상세 패널의 동시 확인 흐름을 유지합니다.
- 위험 시설 목록의 정보 위계를 정리하고, `DrainRiskListItem` 메모화와 선택 콜백 안정화로 선택 변경 시 불필요한 item 갱신을 줄였습니다.
- 날짜 표기를 `yyyy-mm-dd hh:mm:ss` 형식의 공통 formatter로 통일했습니다.
- 목록·상세 패널·이력 영역에 커스텀 스크롤바와 긴 텍스트 처리 규칙을 적용했습니다.

### 테마와 상세 화면 개선

- 시스템/라이트/다크 순환 테마를 도입하고 초기 테마 스크립트와 토글의 hydration 불일치를 수정했습니다.
- 상세 화면의 카드·탭·이력·빈 상태까지 다크 테마를 적용했습니다.
- 상세 화면의 모바일 여백, 시설 주소 줄바꿈, 지도 높이, 유량 단위를 정리했습니다.

### CCTV·지도·긴급 알림

- CCTV 확대 아이콘을 실제 모달 동작으로 연결했습니다.
  - 상세 화면과 대시보드 요약 패널에서 동작합니다.
  - 닫기 버튼, 배경 클릭, Escape 키로 닫을 수 있습니다.
- 지도 마커 클러스터는 시설 수가 30개 이상일 때만 활성화합니다.
  - 소규모 지도에서 혼란을 주던 클러스터 안내 문구는 제거했습니다.
- `danger` WebSocket 이벤트를 헤더 긴급 알림으로 표시합니다.
  - 동일 시설의 반복 이벤트는 시설 ID 기준 한 항목으로 병합해 최신 판단 문구와 시각만 갱신합니다.
  - 알림 목록에서 상세 페이지 이동, 개별 읽음, 모두 읽음 처리를 제공합니다.

## 리뷰 포인트

- 긴급 알림은 외부 발송이 아닌 화면 내 알림입니다. 위험 기준·최대 개수·시설별 병합 기준은 `lib/urgent-alert-policy.ts`를 검토해 주세요.
- 반복 `danger` 이벤트는 기존 항목을 갱신하도록 설계했습니다. 운영 시 재알림 간격(cooldown)이 필요하면 정책 파일에 추가할 수 있습니다.
- 30개는 클러스터 기능을 켜는 시작 기준입니다. 실제 시설 좌표의 분포를 확인한 뒤 `minClusterSize`, `gridSize`, 대표 위험도 색상 정책을 조정해야 합니다.
- production build 실패는 코드 오류가 아닌 `.next` 파일 잠금입니다. 개발 서버를 종료한 깨끗한 환경에서 다시 실행해야 합니다.
