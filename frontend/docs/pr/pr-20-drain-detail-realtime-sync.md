## PR 제목

[fix] 상세 화면 실시간 상태 데이터 동기화

## 작업 내용

상세 화면이 대시보드와 같은 WebSocket 상태 이벤트를 받아 최신 관제 데이터를 즉시 표시하도록 개선했습니다.

- 시설별 `DRAIN_STATUS_UPDATED` 이벤트를 전역 store에 보관합니다.
- 현재 열려 있는 시설의 상세 화면만 해당 상태 이벤트를 병합합니다.
- 상태 이벤트의 수위, 유속, 막힘률, 위험 점수, 최종 판단을 상세 데이터에 반영합니다.
- 센서 차트의 최신 포인트와 위험 이력을 중복 없이 갱신합니다.
- 기존 YOLO/XGBoost 개별 이벤트의 분석 결과·이력 갱신은 유지합니다.
- WebSocket 재연결 뒤에는 기존 화면을 비우지 않고 상세 데이터를 백그라운드에서 보정합니다.
- 이전 상세 조회 응답이 더 최신 WebSocket 이벤트 상태를 덮어쓰지 않도록 요청 순서를 확인합니다.

## 변경 전/후

| 항목 | 변경 전 | 변경 후 |
| --- | --- | --- |
| 상세 상태 | 대시보드 목록만 최신 이벤트를 반영 | 상세 화면도 같은 상태 이벤트를 즉시 병합 |
| 센서 차트 | 최초 상세 조회 데이터 유지 | 이벤트의 수위·유속으로 최신 포인트 추가 |
| 위험 이력 | 최초 조회값 또는 일부 개별 이벤트만 반영 | 최종 상태 이벤트마다 최신 항목 추가 |
| 소켓 재연결 | 상세 화면의 유실 데이터 보정 없음 | 재연결 후 백그라운드 보정 조회 |
| 이전 요청 경합 | 늦게 도착한 상세 API 응답이 최신 상태를 덮을 수 있음 | 요청 순서 확인으로 오래된 응답 무시 |

## 리뷰 포인트

- `DRAIN_STATUS_UPDATED`에 없는 CCTV 이미지 URL과 YOLO 신뢰도는 마지막 정상 값을 유지합니다.
- 현재 상세 페이지는 이벤트의 수위·유속을 차트에 추가합니다. 데이터 제공 순서가 바뀌는 경우 차트 시간 정렬 정책을 별도로 검토해야 합니다.
- 소켓 재연결 보정은 초기 연결이 아닌 재연결 시에만 실행되며, 기존 화면을 로딩 상태로 바꾸지 않습니다.
