## PR 제목

[refactor] 모니터링 대시보드 UI/UX 고도화

## 작업 내용

### 관제 레이아웃과 라이트 모드 표면

- 대시보드를 화면 높이를 채우는 구조로 정리하고, 지도·위험 목록·상세 패널의 데스크톱 높이 기준을 통일했습니다.
- 긴 목록과 상세 데이터는 페이지 전체가 아니라 각 패널 내부에서 스크롤하도록 조정했습니다.
- 라이트 모드의 페이지·빈 여백 배경을 전역 배경 토큰으로 통일하고, 목록 항목의 이중 테두리와 과도한 그림자를 줄였습니다.

### 상세 패널과 센서 데이터 가독성

- 좁은 상세 패널에서 상태는 인라인 배지로 압축하고, 긴 판정 결과는 전체 폭의 문장 영역으로 재배치했습니다.
- 주소 정보는 라벨을 고정하고 값 영역이 자연스럽게 줄바꿈되도록 수정해 글자 단위의 세로 분리를 방지했습니다.
- 센서 요약 카드는 중간 폭에서 2열을 사용하며, 수치·단위·상태 배지에 줄바꿈 방지를 적용했습니다.
- 센서 차트, tooltip, 요약 카드는 다크 모드에서 짙은 표면을 사용하고 grid/axis 색상은 테마 토큰으로 분리했습니다.

### 지도와 헤더 상호작용

- 공용 지도에 `dashboard`와 `detail` variant를 도입했습니다.
  - 상세 지도는 카드의 `220~260px` 높이를 따르므로 기존 공용 지도 `420px` 최소 높이로 인한 overflow가 발생하지 않습니다.
- 메뉴와 사용자 버튼은 기능 구현 전에도 클릭 피드백을 제공합니다.
  - 버튼 기준 안내 팝오버가 2.5초간 표시됩니다.
  - 바깥 클릭, Escape, 다른 버튼 클릭 시 닫힙니다.

## 리뷰 포인트

- `RiskMap`의 높이 variant는 현재 `dashboard`와 `detail` 두 경우만 제공한다. 다른 작은 지도 화면을 추가할 때는 `detail` variant를 재사용하거나 높이 정책을 명시적으로 확장해야 한다.
- 다크 차트의 선·임계선·축 라벨 대비는 실제 다크 모드에서 검토해 주세요.
- 센서 요약은 `2xl` 이상에서 4열, 그보다 좁은 폭에서는 2열이다. 실제 운영 데이터의 긴 단위/수치에서도 줄바꿈이 자연스러운지 확인이 필요하다.
