## PR 제목

[feat] 상세 센서 차트 이중 Y축 및 이미지 사용처 점검

## 작업 내용

- 상세 센서 차트에서 수위와 유속을 독립적인 Y축으로 분리했습니다.
    - 수위: 왼쪽 축, cm, 기본 범위 0~25
    - 유속: 오른쪽 축, m/s, 기본 범위 0~1.5
- 각 데이터 계열의 선·축·범례 색상을 일치시켜 축과 값을 쉽게 구분할 수 있게 했습니다.
- 수위 또는 유속이 기본 범위를 초과하면 해당 축 상한을 자동으로 확장해 값이 잘리지 않게 했습니다.
- 센서 이력 0건은 데이터 없음 상태, 1건은 점과 추세 데이터 부족 안내, 2건 이상은 추세선으로 구분했습니다.
- `frontend/public/images`의 metaimage 2개와 placeholder 5개를 점검했고, 모두 코드에서 참조 중이라 삭제하지 않았습니다.

## 변경 흐름

```text
센서 이력 0건
  → 측정 이력 없음 상태

센서 이력 1건
  → 수위·유속 점 표시
  → 추세 데이터 부족 안내

센서 이력 2건 이상
  → 수위: 왼쪽 Y축(cm)
  → 유속: 오른쪽 Y축(m/s)
```

- 상세 화면에서 수위가 약 20cm, 유속이 약 0.9m/s일 때 두 선이 각각의 축에서 읽기 쉬운지 확인합니다.
- 수위 25cm 또는 유속 1.5m/s를 넘는 값에서 축 상한이 확장되고 데이터가 잘리지 않는지 확인합니다.
