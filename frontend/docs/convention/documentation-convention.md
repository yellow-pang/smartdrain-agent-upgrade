# Frontend Documentation Convention

SmartDrain frontend 문서화 기준을 정리하기 위한 뼈대 문서입니다. 상세 양식은 사용자가 Notion 내용을 옮겨 추가할 예정입니다.

## 1. 목적

작업 계획, 완료 기록, PR 요약을 일관된 흐름으로 남기기 위한 문서입니다.

문서화를 통해 코드 변경 전 작업 범위와 변경 규칙을 미리 확인하고, 범위를 벗어난 작업을 사전에 조정할 수 있도록 합니다.

완료 기록을 통해 변경 내용과 후속 작업을 빠르게 추적할 수 있도록 합니다.

PR 문서를 통해 리뷰에 필요한 내용을 빠르게 정리하고, `docs/pr`에서 PR 기록을 추적할 수 있도록 합니다.

## 2. 적용 범위

이 문서는 `/frontend/docs/` 아래의 작업 문서에 적용합니다.

## 3. plan 문서 작성 기준

계획 문서는 `docs/plans/plan-XX-작업명.md` 경로에 작성합니다. 중간 이상 작업을 시작하기 전에 작업 범위와 접근 방식을 정리합니다.

최소한 무엇을 할지 어떤 내용이 변경될 것인지 사용자가 꼭 확인해야할 내용과 추천 방향을 포함한다.

정해진 양식은 없다.

## 4. step 문서 작성 기준

작업 완료 문서는 `docs/steps/step-XX-작업명.md` 경로에 작성합니다. 실제 변경 내용과 구현 흐름을 다음 작업자가 이해할 수 있게 기록합니다.

정해진 양식은 없다.

사용자가 문서를 보고 어떤 부분이 바뀌었는지 어떤 흐름으로 코드가 흘러가는지 어떻게 변경되었는지 앞으로 무엇을 해야할지를 알 수 있도록 작성합니다.

표와 흐름도를 추가하여 가독성을 높인다.

## 5. PR 문서 작성 기준

PR 요약 문서는 `docs/pr/pr-XX-작업명.md` 경로에 작성합니다. 변경 요약, 검증 결과, 리뷰 포인트를 간단히 정리합니다.

```markdown
### **PR 작성 규칙**

**☑️ PR 제목**
[타입] #제목 (이슈 번호를 포함하면 자동 링크가 걸려 편리함)
ex) [feat] #API 구현

**☑️ PR 내용**

## 작업 내용

<!-- 어떤 변경 사항이 있었는지 주요 내용을 적어주세요. -->

- Room 엔티티와 Participant 엔티티 간 1:N 관계 설정
- 방 생성 시 고유한 6자리 URL Slug 생성 로직 추가
- NestJS ValidationPipe를 이용한 입력값 검증 적용

(## 스크린샷 / 테스트 결과)
(<!-- API 응답 결과(Postman/Swagger)나 실행 결과 스크린샷을 첨부해주세요. -->

- `POST /rooms` 성공 응답 확인 완료)

## 비고
```

`docs/pr/01_v0_frontend_pr.md` 문서가 있으면 해당 문서의 흐름을 참고합니다.

## 6. 변경 전/후 기록 방식

UI, 데이터 구조, 흐름이 바뀐 경우 변경 전과 변경 후를 구분해 적습니다.

## 7. 검증 결과 기록 방식

실행한 명령어, 결과, 실패 원인, 남은 리스크를 구분해서 기록합니다.

## 8. 한글 문서 작성 기준

문서는 기본적으로 한국어로 작성하고, 초보 개발자가 이어서 작업할 수 있게 용어를 풀어서 설명합니다.

## 9. 한글 Conventional Commit 메시지 작성 기준

작업 마지막에는 한글 Conventional Commit 메시지 제목과 내용을 제안합니다.

## 10. 추후 보완 예정 항목

문서 양식이나 기록 방식 중 아직 정하지 않은 항목을 모아둡니다.
