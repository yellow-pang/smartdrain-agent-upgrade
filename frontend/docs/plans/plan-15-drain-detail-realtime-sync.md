# 15 배수 시설 상세 화면 실시간 동기화 계획

## 1. 작업 개요

| 항목 | 내용 |
| --- | --- |
| 현재 브랜치 | `fix/drain-detail-realtime-sync` |
| 작업 규모 | 중간 작업 — 상세 화면의 WebSocket 이벤트 병합과 재연결 보정 흐름을 보완한다. |
| 최종 목표 | 상세 화면을 열어 둔 상태에서 분석 완료 이벤트가 오면 상태 색상뿐 아니라 수위, 유속, 막힘률, 위험 점수, 최종 판단, 센서/위험 이력을 즉시 갱신한다. |
| 작업 범위 | `/frontend` 내부의 상세 화면, 전역 실시간 이벤트 상태, 상세 데이터 상태 |
| 비목표 | 백엔드 WebSocket/API 계약 변경, 새 DB 테이블/컬럼, 새 패키지, 라우팅 변경, 위험도 계산 규칙 변경 |

## 2. 이번 승인 범위

이번 구현은 프론트엔드만 수정한다.

- 기존 `YOLO_RESULT_UPDATED`, `XGBOOST_RESULT_UPDATED`, `DRAIN_STATUS_UPDATED` 이벤트를 상세 화면에 즉시 병합한다.
- 이벤트마다 상세 API를 다시 호출하지 않는다. 따라서 전체 로딩이나 화면 깜빡임이 없다.
- WebSocket 재연결 뒤에만 최신 API 데이터를 백그라운드로 다시 읽어, 연결이 끊긴 동안 놓친 정보를 보정한다. 기존 화면은 유지한다.
- Swagger 입력 경로와 백엔드 이벤트 발행 차이는 이번에 수정하거나 검증하지 않는다.
- 프론트 변경 후 AI 기반 통합 테스트로 실제 이벤트 수신과 상세 갱신을 확인한다. 그 결과 이벤트 payload 또는 발행 경로의 보완이 필요할 때만 백엔드 작업을 별도 계획으로 제안한다.

## 3. 현재 문제와 목표 흐름

```text
분석 결과 저장
  ↓
DRAIN_STATUS_UPDATED WebSocket 이벤트
  ↓
RealtimeDrainSync가 대시보드 시설 목록 캐시를 갱신
  ↓
메인 화면: 최신 수치 반영
상세 화면: 최초 API 조회의 detailData를 주로 유지 → 차트·AI 탭·이력이 오래된 값으로 남음
```

상세 페이지에는 `DRAIN_STATUS_UPDATED`를 병합하는 함수가 선언되어 있지만 실제 수신 흐름에 연결되어 있지 않다. 따라서 다음 데이터가 갱신되지 않는다.

- 센서 차트와 현재 수위/유속 요약
- 위험 점수, 상태, 최종 판단 문구
- 최신 위험 이력
- YOLO/XGBoost 이벤트가 제공하는 분석 결과와 이력

목표는 대시보드와 상세 화면이 같은 이벤트를 각각의 표시 목적에 맞게 즉시 소비하는 것이다.

```text
YOLO_RESULT_UPDATED ─────→ 상세: CCTV/YOLO 분석 카드와 이력 갱신
XGBOOST_RESULT_UPDATED ──→ 상세: XGBoost 분석 카드와 이력 갱신
DRAIN_STATUS_UPDATED ────→ 대시보드: 시설 목록 요약 갱신
                         └→ 상세: 상태·수위·유속·막힘률·차트·위험 이력 갱신
```

## 4. 권장 구현 방향

**대시보드와 같은 이벤트 기반 병합을 상세 화면의 `detailData`에도 적용한다.**

| 항목 | 현재 | 변경 후 |
| --- | --- | --- |
| 상태·수위·유속·막힘률 | 목록 캐시에서만 일부 반영 | `DRAIN_STATUS_UPDATED` 값으로 상세 상태도 즉시 병합 |
| 센서 차트 | 최초 조회값 유지 | 이벤트 수위·유속으로 최신 포인트 추가 |
| 위험 이력 | 최초 조회값 또는 일부 이벤트 값 유지 | 최종 상태 이벤트마다 최신 항목을 중복 없이 추가 |
| YOLO/XGBoost 카드 | 개별 이벤트가 와도 상세 핵심 상태와 분리 | 기존 개별 이벤트 결과를 상세 분석/이력에 병합 |
| 소켓 단절 | 누락된 이벤트를 복구하지 못함 | 재연결 후에만 조용히 API 보정 조회 |

현재 최종 상태 이벤트에는 CCTV 이미지 URL과 YOLO 신뢰도 같은 세부값은 포함되지 않는다. 이 값들은 이번 프론트 변경에서 추측하지 않고 마지막 정상 값을 유지한다. AI 기반 통합 테스트에서 실제 요구가 확인되면 백엔드 이벤트 계약 보완을 별도 작업으로 검토한다.

## 5. 구현 단계

1. 전역 store에 시설별 마지막 `DRAIN_STATUS_UPDATED` 이벤트를 저장한다.
2. 상세 페이지가 현재 시설의 상태 이벤트만 구독하도록 연결한다.
3. 상태 이벤트의 수위·유속·막힘률·판정·최근 시각을 `detailData.drain`에 즉시 병합한다.
4. 상태 이벤트에서 센서 차트 포인트, XGBoost 최신 결과, 위험 이력 항목을 만들고 중복 없이 갱신한다.
5. 기존 `YOLO_RESULT_UPDATED`, `XGBOOST_RESULT_UPDATED` 병합 흐름과 충돌하지 않도록 결과 ID/시각 기준으로 이력을 정리한다.
6. WebSocket 재연결 시에만 상세 API를 백그라운드에서 다시 읽고, 늦게 끝난 이전 요청이 최신 이벤트 상태를 덮어쓰지 않게 한다.

## 7. 검증 계획

| 검증 | 기대 결과 |
| --- | --- |
| `pnpm lint` | 새 ESLint 오류 없음 |
| `pnpm build` | Next.js production build 통과 |
| AI 기반 통합 테스트 | 전체 페이지 로딩/깜빡임 없이 상태, 수위, 유속, 막힘률, 위험 점수, 판단 문구, 센서/위험 이력이 갱신 |
