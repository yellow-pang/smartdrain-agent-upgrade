## 2. 확인한 맥락과 문제

프로젝트의 관리자 대시보드는 지도, 위험 시설 목록, 상세 정보를 통해 우선 확인 대상을 빠르게 찾는 것이 핵심이다. 대시보드 현황은 이 흐름에서 전체 상황과 위험 시설 수를 즉시 알려 주는 요약 정보이므로 제거하지 않고 유지한다.

현재 `DashboardSummaryBar`는 넓은 화면에서 제목·업데이트 시각 묶음과 5개 수치 카드 묶음을 `justify-between`으로 양끝에 배치한다. 이 때문에 두 묶음 사이가 과도하게 비어 보인다. 또한 모든 상태 카드를 동일한 크기로 다루어, 관제 화면에서 가장 먼저 확인해야 할 위험 상태가 전체·양호·판단불가와 같은 무게로 보인다. 5개 고정 항목이 2열 또는 3열로 바뀔 때 마지막 행의 남는 칸도 생긴다.

| 화면 구간 | 현재 문제 | 개선 목표 |
| --- | --- | --- |
| 데스크톱 | 제목과 상태 카드가 양끝으로 분리되어 중앙이 넓고 비어 보임 | 상태 카드가 남은 가로 폭을 활용하도록 해 요약 영역의 정보 밀도를 높임 |
| 모바일 2열 | 다섯 번째 `판단불가` 카드 오른쪽에 빈 칸이 생기며, 전체 현황의 기준점이 약함 | `전체`를 한 줄 전체로 보여 기준값을 먼저 제시하고, 나머지 상태는 2열로 비교 |
| 태블릿 3열 | 네 번째·다섯 번째 카드 뒤에 한 칸이 비어 보이며, 위험 상태가 다른 상태와 동일한 크기임 | `위험`을 2열 폭으로 확장해 즉시 확인 대상을 강조하고, 나머지 상태는 보조 정보로 균형 있게 배치 |

## 3. 권장 구현 방향

**대시보드 현황은 유지하되, 단순히 빈칸을 채우지 않고 관제 우선순위에 따라 카드 크기를 다르게 한다.**

대시보드에서 `전체`는 현재 관리 대상 규모를 이해하는 기준값이고, `위험`은 즉시 확인해야 할 우선 대상이다. `주의`는 다음 확인 대상이며, `양호`와 `판단불가`는 운영 상황을 보완하는 정보다. 따라서 작은 화면에서도 모든 값을 동일한 카드로 나열하지 않고, 전체 현황과 위험 상태를 먼저 눈에 들어오게 한다.

| 화면 크기 | 제목/상태 카드 배치 | 상태 카드 배치 | 기대 효과 |
| --- | --- | --- | --- |
| 모바일 (`< 640px`) | 세로 배치 | `전체`는 2열 전체를 사용하고, 위험·주의·양호·판단불가는 2열로 배치 | 관리 대상 규모를 먼저 파악한 뒤 상태별 분포를 비교할 수 있다. 5개 항목도 1+2+2로 빈칸 없이 정렬된다. |
| 태블릿 (`640px ~ 1023px`) | 세로 배치 | 첫 행은 `전체` 1열 + `위험` 2열, 둘째 행은 주의·양호·판단불가 3열 | 위험 상태의 면적과 시선 우선순위를 높이면서 모든 행을 채운다. |
| 데스크톱 (`>= 1024px`) | 제목 영역 + 남은 폭의 상태 카드 영역 | `전체` 1단위, `위험` 2단위, 주의·양호·판단불가 각 1단위의 6단위 grid | 제목과 수치 사이의 과도한 공백을 줄이고, 위험 상태를 넓게 보여 관제 우선순위를 유지한다. |

구현 시 실제 수치 카드와 `DashboardSummarySkeleton`이 같은 grid·span 규칙을 사용하게 한다. 데이터 로딩 중과 완료 후의 카드 위치가 달라지지 않도록 하기 위함이다. 상태 색상은 기존 기준(위험 red, 주의 amber, 양호 green, 판단불가 slate)을 그대로 유지하며, 위험 카드의 강조는 색상뿐 아니라 넓이와 명확한 레이블로 제공한다.

## 4. 구현 단계

1. `DashboardSummaryBar`의 데스크톱 컨테이너를 제목의 고정/최소 폭 영역과 가변 폭 상태 카드 영역으로 정리한다.
2. 상태 카드의 중요도 메타데이터 또는 명시적인 class를 통해 화면별 span을 적용한다.
   - 모바일에서는 `전체` 카드가 전체 행을 사용하고, 나머지 4개 카드는 2열로 배치한다.
   - 태블릿에서는 `전체` 1열과 `위험` 2열을 첫 행에 배치하고, 주의·양호·판단불가는 다음 행에 균등 배치한다.
   - 데스크톱에서는 6단위 grid에서 `위험`만 2단위를 사용하고 나머지 카드는 1단위를 사용한다.
3. `DashboardSummarySkeleton`에도 동일한 그리드·span 규칙을 적용한다.
4. 긴 최신 업데이트 시각이 제목 영역을 넘지 않도록 기존 줄바꿈 규칙을 유지하고, 카드의 라벨·수치가 줄바꿈 또는 겹침 없이 보이는지 확인한다.
5. 지도·위험 목록·상세 패널의 데이터 선택과 기존 반응형 전환은 변경하지 않는지 회귀 확인한다.

## 6. 사용자 확인이 필요한 사항

| 항목 | 권장안 | 이유 |
| --- | --- | --- |
| 대시보드 현황 유지 여부 | **유지** | 위험 시설 수와 전체 분포는 지도·목록을 보기 전 우선순위를 빠르게 판단하는 데 필요한 요약 정보다. |
| 정보 중요도 | **전체는 기준값, 위험은 최우선, 주의는 다음 우선, 양호·판단불가는 보조 정보** | 같은 크기의 카드 나열보다 실제 관제 판단 순서를 화면에 반영한다. |
| 데스크톱 카드 폭 | **제목은 필요한 최소 폭만 유지하고, 위험 카드만 다른 카드보다 넓게 사용** | 양끝 정렬로 생기는 중앙 공백을 줄이고 즉시 확인 대상인 위험을 강조한다. |
| 모바일 2열의 전체 카드 | **`전체`를 전체 행 너비로 표시** | 첫 시선에서 관리 대상 규모를 파악하고, 이어지는 두 줄에서 상태별 분포를 비교할 수 있다. |
| 태블릿 3열의 우선순위 | **첫 행에 `전체` 1열 + `위험` 2열 배치** | 위험 상태를 넓게 표시해 대응 우선순위를 분명히 하면서 행의 빈칸도 만들지 않는다. |
| 적용 범위 | **현황 요약 바에 한정** | 현재 피드백의 원인을 직접 해결하고 지도·목록·상세 관제 레이아웃의 검증 범위를 불필요하게 넓히지 않는다. |

위 권장안대로 진행하면 별도 디자인 방향 결정 없이 구현할 수 있다. 현황을 제거하거나 상태 항목 수를 줄이는 방향을 원한다면 API 응답·운영 우선순위 표시 기준까지 함께 재검토해야 하므로, 그 경우에는 구현 전에 범위를 다시 정한다.

## 8. 리스크와 대응

| 리스크 | 대응 |
| --- | --- |
| 태블릿 폭에서 최신 업데이트 시각이 길어질 수 있음 | 제목 영역과 카드를 세로 배치하고 날짜 텍스트의 줄바꿈을 허용한다. |
