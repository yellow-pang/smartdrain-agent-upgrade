# 16 모바일 긴급 알림 패널 레이아웃 수정 계획

## 1. 작업 개요

| 항목 | 내용 |
| --- | --- |
| 현재 브랜치 | `fix/mobile-urgent-alert-layout` |
| 작업 규모 | 중간 작업 — 공용 헤더의 긴급 알림 패널 위치 정책과 모바일 반응형 표시를 조정한다. |
| 최종 목표 | 360px 폭을 포함한 모바일 화면에서 긴급 알림 제목·설명·목록이 잘리지 않고, 알림 버튼을 눌렀을 때 화면 안에서 읽기 쉬운 너비로 표시된다. |
| 작업 범위 | `/frontend/components/app-header.tsx`의 긴급 알림 패널 위치·너비·모바일 스크롤 처리 및 관련 frontend 문서 |
| 비목표 | 알림 데이터 구조, WebSocket 이벤트 처리, 알림 읽음 처리, 헤더 메뉴/사용자 기능, 새 패키지 설치 |

## 2. 확인한 문제

현재 긴급 알림 패널은 알림 버튼을 감싼 `relative` 컨테이너 안에서 다음과 같이 배치된다.

```text
알림 버튼 기준 relative 컨테이너
  └─ absolute right-0
      └─ width: min(22rem, 100vw - 1.5rem)
```

패널의 오른쪽 끝은 알림 버튼에 맞지만, 모바일 헤더 오른쪽에는 사용자 버튼·구분선 등의 영역이 남아 있다. 따라서 `360px` 같은 좁은 화면에서는 패널 너비가 화면보다 넓게 왼쪽으로 밀려 제목과 설명의 앞부분이 잘린다.

## 3. 권장 구현 방향

**모바일에서는 뷰포트 기준의 고정 패널, `sm` 이상에서는 현재처럼 알림 버튼 기준 팝오버를 사용한다.**

| 화면 크기 | 위치 정책 | 너비 정책 | 이유 |
| --- | --- | --- |
| 기본 모바일 (`< 640px`) | 헤더 아래의 `fixed` 패널 | 좌우 `0.75rem` 여백을 둔 가변 너비 | 버튼 위치와 무관하게 항상 화면 안에 남는다. |
| 태블릿·데스크톱 (`sm` 이상) | 알림 버튼 기준 `absolute right-0` | 최대 `22rem` | 현재 데스크톱의 팝오버 경험과 버튼 연관성을 유지한다. |

고정 패널은 헤더(`z-30`)보다 위에 표시하되, 기존 알림 패널의 `z-50`을 유지한다. 목록이 길 때는 기존 내부 `max-h` 스크롤을 유지하고, 패널 전체가 작은 화면의 세로 공간을 과도하게 차지하지 않도록 최대 높이도 함께 점검한다.

## 4. 구현 단계

1. `app-header.tsx`의 긴급 알림 패널 class를 모바일 우선으로 변경한다.
   - 모바일: 헤더 아래 `fixed`, 화면 좌우 여백 적용
   - `sm` 이상: 현재의 버튼 기준 `absolute right-0` 복원
2. 패널 헤더의 제목·설명이 좁은 폭에서도 줄바꿈 또는 축소 규칙에 따라 잘리지 않는지 확인한다.
3. 긴급 알림 목록, 빈 상태, `모두 읽음` 버튼이 360px 폭에서 겹치지 않는지 확인한다.
4. 모바일에서 알림을 닫거나 시설 링크를 눌렀을 때 기존 동작이 유지되는지 확인한다.

## 6. 사용자 확인이 필요한 사항

| 항목 | 권장안 | 이유 |
| --- | --- | --- |
| 모바일 패널 위치 | **헤더 아래 화면 고정(`fixed`)** | 버튼 위치에 영향을 받지 않아 어떤 모바일 폭에서도 왼쪽 잘림이 없다. |
| 모바일 가로 여백 | 좌우 `12px` | 360px 화면에서도 읽기 가능한 너비를 확보하면서 화면 경계를 피한다. |
| 데스크톱 동작 | 기존 버튼 기준 팝오버 유지 | 넓은 화면의 현재 상호작용과 레이아웃을 바꾸지 않는다. |
| 긴 알림 목록 | 현재 내부 스크롤 유지 | 헤더와 지도/대시보드 영역을 가리지 않으면서 여러 알림을 확인할 수 있다. |

## 7. 검증 계획

| 검증 | 기대 결과 |
| --- | --- |
| `pnpm lint` | ESLint 오류 없이 통과 |
| 360×740 (Galaxy S8+) | 제목·설명·빈 상태·알림 목록의 왼쪽 잘림 없음 |
| 390×844 (iPhone 12 Pro) | 패널이 헤더 아래 화면 안에 표시되고 오른쪽 아이콘과 겹치지 않음 |
| 768px 이상 | 패널이 알림 버튼 아래 오른쪽에 표시됨 |
| 알림 0건/여러 건 | 빈 상태와 목록 스크롤, 모두 읽음 버튼이 정상 동작 |
| 링크·닫기 동작 | 시설 링크 이동과 알림 닫힘 동작이 기존과 같음 |

## 8. 리스크와 대응

| 리스크 | 대응 |
| --- | --- |
| fixed 패널이 모바일 대시보드 콘텐츠를 가림 | 헤더 바로 아래에 제한된 높이로 표시하고, 목록은 내부 스크롤을 사용한다. |
| 모바일/데스크톱 위치 전환이 어색함 | `sm` 브레이크포인트에서 위치와 너비 규칙을 함께 전환한다. |
| 긴 시설명·판정 문구가 패널 높이를 늘림 | 기존 `break-words`와 내부 목록 스크롤을 유지해 가로 넘침을 막는다. |
