# 32 긴급 알림 패널 웹 표시 개선 계획

## 작업 목표

웹 화면에서 긴급 알림 패널의 줄바꿈과 표시 개수 체감 문제를 줄이고, 여러 알림이 한 번에 들어와도 관리자가 최신 위험 상태를 먼저 확인할 수 있게 개선한다.

## 문제 정의

| 문제 | 영향 |
| --- | --- |
| 웹 패널 폭이 좁아 `모두 읽음`, 시설명, 상태 배지가 줄바꿈됨 | 관리자가 알림을 빠르게 스캔하기 어렵다. |
| 알림 목록 영역이 낮아 3개 정도만 보임 | 여러 시설이 동시에 위험해질 때 누락처럼 느껴진다. |
| 같은 시설 알림이 갱신될 때 기존 위치를 유지함 | 최신 알림이 아래쪽에 남아 UX가 어색하다. |
| 여러 알림이 한 번에 들어오는 경우 정책이 화면에 드러나지 않음 | 어떤 알림을 먼저 봐야 하는지 불명확하다. |

## 수정 방향

| 항목 | 방향 |
| --- | --- |
| 정렬 | `updatedAt` 기준 최신순으로 렌더링한다. 같은 시설 갱신도 최신 위치로 올라오게 한다. |
| 표시 개수 | store는 기존 정책대로 최대 20개를 유지하고, 패널은 스크롤로 전체 확인하게 한다. |
| 웹 패널 폭 | 데스크톱에서는 패널 폭을 넓혀 주요 문구 줄바꿈을 줄인다. |
| 헤더 | `모두 읽음` 버튼은 줄바꿈되지 않게 하고, 최신순/보관 개수를 명확히 보여준다. |
| 목록 행 | 시설명은 한 줄 truncate, 상태 배지는 고정, 메시지는 최대 2줄로 제한한다. |
| 동시 알림 | 여러 건이 한 번에 들어와도 최신순 목록과 요약 count로 우선순위를 판단한다. |

## 예상 변경 범위

| 파일 | 변경 |
| --- | --- |
| `components/app-header.tsx` | 알림 패널 폭, 헤더, 최신순 정렬, 목록 행 줄바꿈 개선 |
| `store/drain-store.ts` | 같은 시설 알림 갱신 시 최신 항목이 앞으로 오도록 upsert 순서 보강 |
| `docs/steps/step-48-urgent-alert-panel-format.md` | 작업 결과 기록 |
| `docs/pr/pr-37-urgent-alert-panel-format.md` | PR 요약 기록 |

## 검증 계획

| 검증 | 기대 결과 |
| --- | --- |
| `npm.cmd run lint` | 신규 오류 없음 |
| `npm.cmd run build` | 빌드 성공 |
| 웹 알림 패널 | `모두 읽음` 버튼과 시설명/배지가 어색하게 줄바꿈되지 않음 |
| 다중 알림 | 4개 이상 알림도 스크롤로 확인 가능 |
| 최신 갱신 | 최신 알림이 목록 상단에 표시됨 |
