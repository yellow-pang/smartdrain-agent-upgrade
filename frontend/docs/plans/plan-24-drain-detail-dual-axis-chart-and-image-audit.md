# 24 상세 센서 이중 Y축 차트 및 이미지 사용처 점검 계획

## 1. 작업 개요

| 항목 | 내용 |
| --- | --- |
| 작업 브랜치 | `feat/drain-detail-dual-axis-chart` |
| 작업 규모 | 중간 작업 — 상세 차트 UI·데이터 표시 조건과 정적 이미지 참조를 함께 점검한다. |
| 최종 목표 | 수위와 유속을 각각 읽기 쉬운 이중 Y축으로 표시하고, 데이터 수가 부족한 경우 오해를 막는 안내를 제공하며, 실제 사용 중인 이미지를 보존한 상태로 정적 자산 사용처를 정리한다. |
| 이번 범위 | `frontend/components/sensor-trend-chart.tsx`, 필요한 차트 helper·테스트 데이터, `frontend/public/images` 참조 점검, 계획·완료·PR 문서 |
| 비목표 | 차트 라이브러리 교체, API·WebSocket 계약 변경, 루트 `/docs` 수정, 실제 CCTV/S3 저장 방식 변경, 참조 중인 placeholder 삭제 |

## 2. 현재 확인 결과

### 2.1 센서 차트

- `SensorTrendChart`는 수위(`level`, cm)와 유속(`flow`, m/s)을 하나의 `<YAxis>`에 표시한다.
- 수위와 유속은 단위와 일반적인 값 범위가 달라 유속 선이 그래프 하단에 붙어 보인다.
- `yAxisId`가 없어 두 데이터 계열을 독립 축에 연결하지 못한다.
- 데이터가 1건이어도 선을 그리는 별도 조건이나 추세 데이터 부족 안내가 없다.

### 2.2 이미지 자산

`frontend/public/images`의 이미지 7개는 모두 사용 중이다.

| 자산 | 사용처 | 점검 결과 |
| --- | --- | --- |
| `metaimage/smartdrain-icon.png` | 앱·Apple 아이콘 | 사용 중 |
| `metaimage/smartdrain-og-image.png` | Open Graph·Twitter 이미지 | 사용 중 |
| `placeholder/01_placeholder_cctv_feed_unavailable.png` | CCTV 이미지 fallback | 사용 중 |
| `placeholder/02_placeholder_map_unavailable.png` | 지도 데이터 없음 상태 | 사용 중 |
| `placeholder/03_placeholder_realtime_chart_unavailable.png` | 센서 데이터 연결 대기 상태 | 사용 중 |
| `placeholder/04_placeholder_facility_detail_unavailable.png` | 시설 요약·상세 fallback | 사용 중 |
| `placeholder/05_placeholder_thumbnail_unavailable.png` | 썸네일 이미지 fallback | 사용 중 |

현재 기준으로 삭제 가능한 `public/images` 파일은 없다. `/test-snapshots/*`는 별도 테스트 자산 경로로 문서·mock 응답에서 참조되므로, 이번 `public/images` 정리 대상에 포함하지 않는다.

## 3. 권장 구현 방향

### 3.1 이중 Y축

```text
왼쪽 Y축 (yAxisId: waterLevel) : 수위, cm, 기본 0~25
오른쪽 Y축 (yAxisId: flowVelocity) : 유속, m/s, 기본 0~1.5
```

- 수위 선은 왼쪽 축, 유속 선은 오른쪽 축에 각각 연결한다.
- 각 축의 색상은 해당 선·범례와 일치시켜 읽기 쉽게 한다.
- 데이터가 기본 범위를 넘으면 값이 잘리지 않도록 해당 축의 상한을 자동 확장한다. 기본 눈금 기준은 유지한다.
- 툴팁은 값과 단위를 각각 명시한다.

### 3.2 데이터 수별 표시

| 데이터 수 | 권장 표시 |
| --- | --- |
| 0건 | 기존 연결 대기/데이터 없음 상태를 표시한다. |
| 1건 | 선 연결 없이 수위·유속 점만 표시하고 `측정 데이터 1건 — 추세를 표시하려면 추가 측정이 필요합니다.`를 표시한다. |
| 2건 이상 | 각 축에 연결된 선과 점으로 추세를 표시한다. |

### 3.3 이미지 정리

- 이번 작업에서는 삭제하지 않는다. fallback 이미지들은 오류·데이터 없음 상태에서 필요한 사용자 안내 자산이다.
- 새 이미지가 추가되거나 화면이 제거돼 참조가 0건인 파일이 생긴 경우에만, 코드·문서·배포 설정 검색 결과를 목록으로 제시한 뒤 삭제 승인을 받는다.

## 4. 구현 단계

| 단계 | 변경 내용 | 완료 기준 |
| --- | --- | --- |
| 1. 차트 데이터 조건 정리 | 0·1·다건 포인트를 구분하고 화면 문구와 점/선 표시 규칙을 추가한다. | 1건 데이터가 추세선으로 오해되지 않는다. |
| 2. 이중 Y축 적용 | Recharts의 양쪽 `YAxis`와 각 `Line.yAxisId`를 연결하고 축 단위·색상·범위를 설정한다. | 수위와 유속이 각자 축 기준으로 읽힌다. |
| 3. 범위·반응형 검증 | 기본 범위와 초과 데이터 자동 확장, 모바일/데스크톱의 축 라벨·툴팁 겹침을 확인한다. | 높은 수위·낮은 유속에서도 두 변화가 식별된다. |
| 4. 이미지 사용처 재검증 | `public/images` 전체 참조를 다시 검색하고 삭제 후보 유무를 기록한다. | 참조 중인 이미지가 삭제되지 않는다. |
| 5. 검증·기록 | lint/build와 0·1·다건 차트 수동 확인 결과를 step·PR 문서에 기록한다. | 변경 결과와 남은 리스크가 추적 가능하다. |

## 6. 사용자 확인이 필요한 사항

| 확인 항목 | 권장 방향 | 선택에 따른 영향 |
| --- | --- | --- |
| 축 범위 | **수위 0~25 cm, 유속 0~1.5 m/s를 기본으로 하고 초과 값은 자동 확장한다.** | 완전 고정 범위는 이상치를 잘라낼 수 있고, 완전 자동 범위는 화면 간 비교 기준이 달라질 수 있다. |
| 단일 데이터 표시 | **선 없이 점과 데이터 부족 안내 문구를 함께 표시한다.** | 선을 유지하면 실제 추세가 있는 것처럼 보일 수 있다. |
| 데이터 0건 표시 | **현재 연결 대기 fallback을 유지하되, API 성공·빈 이력인 경우에는 데이터 없음 문구를 구분한다.** | 연결 실패와 실제 측정 데이터 부재를 구별할 수 있다. |
| 이미지 삭제 | **이번 작업에서는 삭제하지 않는다.** | 현재 `public/images`의 7개 이미지가 모두 사용 중이므로 삭제하면 fallback·메타데이터가 손상된다. |
| 테스트 스냅샷 | **`/test-snapshots/*`는 별도 테스트 자산으로 유지한다.** | 문서와 mock 응답이 참조하므로 삭제·이동은 백엔드 통합 테스트 범위로 별도 합의가 필요하다. |

## 7. 검증 시나리오

1. 0건, 1건, 2건 이상 센서 이력에서 차트 표시 상태와 문구를 확인한다.
2. 수위 20 cm·유속 0.9 m/s 데이터를 넣어 두 선이 각각의 축에서 읽히는지 확인한다.
3. 수위 25 cm·유속 1.5 m/s 초과 데이터가 잘리지 않고 축 상한이 확장되는지 확인한다.
4. 모바일·데스크톱에서 양쪽 축 라벨과 툴팁이 겹치지 않는지 확인한다.
5. `rg`로 `public/images` 파일별 참조를 확인하고, 참조 중인 이미지가 삭제되지 않았는지 검증한다.
6. `package.json` 기준 `pnpm lint`, `pnpm build`를 실행한다.

## 8. 리스크와 대응

| 리스크 | 대응 |
| --- | --- |
| 두 축이 값의 상관관계를 과장해 보일 수 있음 | 축 라벨·단위·선 색상을 명확히 표시하고 툴팁에서 원값을 제공한다. |
| 고정 축 상한을 초과한 실제 데이터 | 기본 범위를 넘는 경우에만 자동 확장해 데이터 손실을 막는다. |
| 빈 이력과 API 연결 실패 혼동 | fallback 여부와 빈 points 상태를 분리해 적절한 안내를 표시한다. |
| placeholder 삭제 후 오류 화면 손상 | 모든 코드·문서 참조를 확인하고 사용자 승인 없는 삭제를 하지 않는다. |

## 9. 승인 후 첫 작업

승인된 축·데이터 표시 정책을 기준으로 `SensorTrendChart`에 이중 Y축과 단일 포인트 안내를 구현하고, 이미지 자산은 삭제 없이 참조 점검 결과만 작업 완료 문서에 기록한다.
