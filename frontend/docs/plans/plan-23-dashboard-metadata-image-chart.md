# 23 대시보드 메타데이터·최신 CCTV 이미지·센서 차트 개선 계획

## 1. 작업 개요

| 항목        | 내용                                                                                                                                                                      |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 작업 브랜치 | 요청: `feat/dashboard-metadata-image-chart` / 현재 확인값: `feature/frontend`                                                                                             |
| 작업 규모   | 중간 작업 — 메타데이터, 대시보드·상세 화면 이미지 데이터 흐름, 상세 차트 UI가 함께 변경된다.                                                                              |
| 최종 목표   | SmartDrain 서비스 목적에 맞는 검색·공유 메타데이터를 제공하고, 메인과 상세에서 동일한 최신 CCTV 이미지를 일관되게 표시하며, 수위와 유속을 각각 읽기 쉬운 축으로 표시한다. |
| 이번 범위   | `frontend/app/layout.tsx`, 이미지 표시 컴포넌트와 데이터 adapter/mock, `sensor-trend-chart.tsx`, `frontend/public/images`의 신규 이미지 및 사용처 점검, 작업 문서         |
| 비목표      | API 계약 변경, 새 패키지·차트 라이브러리 도입, 이미지 분석 로직 변경, 루트 `/docs` 수정                                                                                   |

## 2. 확인한 기준과 현재 상태

- 프로젝트 문서상 SmartDrain은 CCTV 스냅샷과 수위(cm)·유속(m/s)을 결합해 빗물받이 침수 위험을 관제하는 서비스다. API 명세의 상세 DTO에는 `imageUrl`, `sensorHistory`, `waterLevelCm`, `flowVelocityMps`가 정의돼 있다.
- 현재 전역 메타데이터는 제목·설명은 있으나 `generator: "v0.app"`이 남아 있고, Open Graph·Twitter 공유 정보와 프로젝트 전용 이미지가 없다.
- 상세 화면의 `CctvSnapshotCard`는 분석 이력 배열의 첫 항목(최신 캡처 순서)을 기본으로 표시한다. 상세 데이터가 없으면 CCTV placeholder를 표시한다.
- 메인 대시보드의 `DrainSummaryPanel`은 `imageUrl` prop이 전달되지 않아 항상 시설 placeholder를 기본값으로 표시한다. 그래서 상세에는 실제 이미지가 보이는데 메인에는 다른 이미지 또는 placeholder가 보일 수 있다.
- mock 분석 이력은 첫 항목만 CCTV placeholder이고 이후 항목은 `/test-snapshots/...` 경로를 사용한다. 해당 파일은 현재 `public/images` 목록에서 확인되지 않아 이미지 로드 실패 시 fallback이 발생할 수 있다.
- 현재 센서 차트는 수위와 유속을 단일 Y축에 그린다. 값 범위가 다른 단위를 공유하므로 유속 선이 하단에 붙어 보인다. 데이터가 1건이어도 선 연결을 막는 별도 처리가 없다.

## 3. 권장 구현 방향

### 3.1 메타데이터

- 서비스명은 `SmartDrain`, 제목은 `SmartDrain | 지능형 도시 침수 관리`로 정리한다.
- 설명은 CCTV 스냅샷, 수위·유속 센서, AI 기반 빗물받이 위험 관제를 담아 프로젝트 목적과 일치시킨다.
- `generator`는 제거하고, 기본 URL·Open Graph·Twitter 카드·아이콘을 Next.js Metadata API로 설정한다.
- 사용자가 제공하는 프로젝트 대표 이미지를 Open Graph/Twitter 이미지로 사용한다. 이미지는 `frontend/public/images` 아래에 추가하고, 실제 파일명·비율을 확인한 뒤 경로를 연결한다.

### 3.2 메인·상세 최신 이미지 일관성

이미지 표시 우선순위를 아래처럼 통일한다.

```text
분석 이력의 최신 imageUrl
→ 상세 DTO의 imageUrl 또는 yoloResult.imageUrl
→ placeholder
```

- API 데이터가 있는 경우 메인 선택 시설 패널과 상세 CCTV 카드가 같은 최신 이미지 URL을 사용하도록 UI 모델 또는 prop 흐름을 보완한다.
- 이미지 URL이 없거나 로드에 실패한 경우에만 화면 목적에 맞는 placeholder를 사용한다.
- mock 데이터도 같은 우선순위를 검증할 수 있도록 실제 존재하는 정적 테스트 이미지 또는 사용자가 제공할 이미지를 참조한다. mock 이미지 구조를 실제 백엔드 계약으로 확정하지 않는다.

### 3.3 상세 센서 차트

- Recharts는 유지하고, 왼쪽 Y축에 수위(cm), 오른쪽 Y축에 유속(m/s)을 둔다.
- 기본 범위는 수위 `0~25 cm`, 유속 `0~1.5 m/s`로 적용하되, 범위를 초과하는 실제 데이터가 잘리지 않도록 최댓값을 반영해 상한을 확장한다.
- 선과 축·툴팁 라벨의 색상을 일치시키고, 각각의 단위를 명시한다.
- 데이터가 1건이면 선 대신 점만 표시하고, `측정 데이터 1건 — 추세를 표시하려면 추가 측정이 필요합니다.` 안내를 표시한다.
- 데이터가 0건인 경우 기존 연결 대기 placeholder 흐름을 유지한다.

## 4. 구현 단계

| 단계                | 변경 내용                                                                                                 | 완료 기준                                                                   |
| ------------------- | --------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------- |
| 0. 입력·사용처 점검 | 사용자가 넣은 `public/images` 파일, 기존 placeholder 및 test snapshot 참조를 목록화한다.                  | 신규 이미지의 용도와 기존 이미지의 실제 참조 여부를 확인한다.               |
| 1. 메타데이터 정비  | 전역 제목·설명·공유 카드·아이콘 정보를 프로젝트 기준으로 교체한다.                                        | 브라우저 문서 제목과 메타 태그에 v0 흔적 없이 SmartDrain 정보가 표시된다.   |
| 2. 이미지 흐름 정렬 | 최신 분석 이미지 선택 helper/데이터 모델을 만들고 대시보드와 상세에 동일하게 적용한다.                    | 동일 시설에서 메인과 상세가 같은 최신 이미지 또는 같은 fallback을 표시한다. |
| 3. 차트 개선        | 이중 Y축, 단일 데이터 포인트 표시, 축 범위와 안내 문구를 구현한다.                                        | 유속 변화가 수위와 독립적으로 읽히며 0·1·다건 데이터가 모두 정상 표시된다.  |
| 4. 정리·검증        | 실제 참조되지 않는 이미지 후보를 확인하고 승인된 파일만 삭제한다. lint/build와 주요 화면 확인을 수행한다. | 삭제 대상이 근거와 함께 확정되고, 검증 결과를 step 문서에 남길 수 있다.     |

## 7. 검증 시나리오

1. `package.json` 기준으로 `pnpm lint`, `pnpm build`를 실행한다.
2. 메인 대시보드에서 시설 선택 시 최신 분석 이미지가 표시되는지, 상세 페이지의 첫 CCTV 이미지와 URL이 같은지 확인한다.
3. 이미지 없음·이미지 로드 실패·분석 이력 없음에서 placeholder가 의도대로 표시되는지 확인한다.
4. 센서 이력 0건, 1건, 여러 건에서 차트/안내 문구와 왼쪽 수위·오른쪽 유속 축을 확인한다.
5. 모바일과 데스크톱에서 차트 축 라벨·툴팁·이미지가 겹치지 않는지 확인한다.
6. 브라우저 개발자 도구 또는 메타데이터 검사 도구로 title, description, Open Graph/Twitter image 경로를 확인한다.

## 8. 리스크와 대응

| 리스크                            | 대응                                                                                                                   |
| --------------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| API 목록 응답에 이미지 URL이 없음 | 데이터 계약을 임의로 바꾸지 않고, 최신 분석 조회 가능 여부를 확인한 뒤 이미지가 없으면 명시적으로 fallback을 사용한다. |
| 이미지가 아직 제공되지 않음       | 메타데이터 코드의 실제 이미지 경로 연결과 미사용 파일 삭제는 이미지 투입 이후로 진행한다.                              |
| 고정 축 범위를 넘는 센서 값       | 기본 범위는 유지하되 실제 최대값에 맞춰 축 상한을 확장한다.                                                            |
| 1건 데이터를 추세로 오해          | 선을 숨기고 점·안내 문구로 데이터 부족을 표시한다.                                                                     |
| placeholder 삭제로 fallback 손상  | 각 placeholder 상수와 모든 import 사용처를 확인하고, 삭제 전 사용자 승인을 받는다.                                     |

## 9. 승인 후 첫 작업

브랜치 상태와 이미지 제공 경로를 확인한 후, 전역 메타데이터를 SmartDrain 기준으로 교체하고 최신 이미지 선택 흐름 및 이중 Y축 차트를 구현한다.
