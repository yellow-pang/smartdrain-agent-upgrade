# 27 상세 분석 요약 컴포넌트화 및 데스크톱 밀도 개선

## 작업 목표

상세 화면의 `현재 분석 요약` 카드가 넓은 웹 화면에서도 2×2 배치로 남아 불필요한 빈 영역이 커지는 문제를 개선했다. 막힘 정도, 수위, 유속, 최종 판단을 관제 대시보드에서 한 번에 비교할 수 있도록 일반 데스크톱 폭부터 한 줄 4열로 배치했다.

## 변경 내용

| 구분 | 변경 전 | 변경 후 |
| --- | --- | --- |
| 컴포넌트 위치 | `app/drains/[id]/page.tsx` 내부 `AnalysisSummaryCard` | `components/drain-detail/analysis-summary-card.tsx` |
| 넓은 화면 배치 | 화면 폭과 무관하게 2열×2행 | `lg`(1024px) 이상에서 4열×1행 |
| 태블릿/모바일 | 2열 고정 | `sm` 이상 2열, `sm` 미만 1열 |
| 최종 판단 텍스트 | 넓은 카드에서 여백이 크고 긴 텍스트의 표시 규칙이 분산 | `min-w-0`, `break-keep`, 고정 line-height로 카드 안에서 읽기 좋게 줄바꿈 |
| 누락 막힘률 | 기존 문자열 보간 시 `null%` 가능성 | 값이 없으면 `-`를 표시 |

## 화면 구성

```text
일반 웹 화면 (1024px 이상)
[ 막힘 정도 ] [ 수위 ] [ 유속 ] [ 최종 판단 ]

태블릿
[ 막힘 정도 ] [ 수위 ]
[ 유속      ] [ 최종 판단 ]

모바일
[ 막힘 정도 ]
[ 수위      ]
[ 유속      ]
[ 최종 판단 ]
```

최종 판단 카드의 시간은 한 줄을 넘길 경우 말줄임표로 처리하고, 전체 값은 `title` 속성으로 확인할 수 있게 했다. 핵심 판단 문구는 줄바꿈하되 단어 단위로 유지해 글자가 세로로 쪼개지지 않게 한다.

## 유지한 동작

- 최신 YOLO 결과의 막힘률, 최신 XGBoost 결과의 판단 문구·평가 시각 선택 규칙을 유지한다.
- 위험 상태별 색상, 막힘률 progress bar, API 데이터·WebSocket 갱신 흐름은 변경하지 않았다.
- 상세 route의 API 요청·이벤트 병합 helper는 route에 그대로 둔다.

## 검증 결과

| 명령어 | 결과 | 비고 |
| --- | --- | --- |
| `pnpm.cmd lint` | 통과 | 오류 0건, 기존 `components/fallback-image.tsx`의 `<img>` 경고 1건 유지 |
| `pnpm.cmd build` | 통과 | Next.js production build 성공. `/` 정적 생성, `/drains/[id]` 동적 route 확인 |

## 남은 수동 확인

브라우저 수동 점검은 현재 실행하지 않았다. 다음 통합 점검에서 1024px, 1280px, 1600px 이상 폭에서 요약 카드가 4열 한 행으로 표시되는지, 768px·360px에서 카드와 최종 판단 문구가 넘치지 않는지 확인한다.
