# 39 모바일 선택 시설 인라인 요약 및 상세 탐색 UI 개선 결과

## 작업 결과

모바일 대시보드에서 선택 시설의 요약과 상세 이동을 목록 전체 끝이 아닌 **선택한 목록 항목 바로 다음**에 표시하도록 변경했다. 긴 목록에서도 선택·핵심 수치 확인·상세 이동을 한 화면 구간에서 이어 갈 수 있다.

상세 화면의 XGBoost 탭은 여섯 항목을 빈칸 없이 2열×3행으로 정리했다. 날짜와 긴 판단 문구가 카드 폭 또는 높이를 과도하게 밀지 않도록 말줄임 규칙을 적용했다.

모바일 상세 화면에는 콘텐츠를 가리는 전체 폭 복귀 바 대신, 충분히 아래로 스크롤했을 때만 표시되는 작은 `대시보드` 플로팅 칩과 상단 이동 버튼을 추가했다.

## 변경 흐름

```text
모바일 대시보드
목록 항목 선택
  -> selectedDrainId 갱신
  -> 선택 항목 바로 뒤에 시설 요약 표시
  -> 상세 정보 페이지로 이동

모바일 상세 화면
긴 콘텐츠를 아래로 스크롤
  -> 480px 이후 상단 이동 버튼 표시
  -> 상단 이동 또는 대시보드 복귀 선택
```

## XGBoost 탭 배치

```text
[ XGBoost Result ID ] [ 위험 상태      ]
[ 참조 Sensor ID    ] [ 참조 YOLO ID  ]
[ 판단 시각         ] [ 최종 판단 문구]
```

- 각 셀은 고정 높이를 사용한다.
- 판단 시각은 한 줄 말줄임표로 표시한다.
- 최종 판단 문구는 최대 두 줄로 제한하고 전체 문구는 `title` 속성으로 확인할 수 있다.
- 상태 배지와 위험도 데이터·API·WebSocket 병합 흐름은 변경하지 않았다.

## 남은 수동 확인

1. 360×740, 375×667, 390×844에서 목록의 첫 시설 기본 선택과 다른 시설 선택 모두 요약 카드가 해당 항목 바로 아래에 표시되는지 확인한다.
2. 모바일 상세 화면에서 전체 폭 복귀 바가 보이지 않고, 마지막 콘텐츠가 안전 영역과 플로팅 빠른 행동에 가려지지 않는지 확인한다.
3. 480px 전후 스크롤에서 `대시보드` 칩과 상단 이동 버튼이 의도한 시점에 나타나며, 터치 영역이 서로 겹치지 않는지 확인한다.
4. XGBoost 탭에서 긴 날짜·판단 문구가 360px, 640px, 1024px 폭에서 셀 밖으로 넘치지 않는지 확인한다.
5. 지도 선택, 목록 선택, WebSocket 갱신 후 선택 시설의 요약·상세 링크·목록 강조가 같은 대상을 가리키는지 확인한다.

## 남은 리스크

- `title` 속성은 마우스 환경에서 전체 값을 쉽게 볼 수 있으나, 모바일에서는 긴 최종 판단 문구를 상세하게 읽을 별도 수단이 없다. 실제 운영 문구가 자주 길다면 추후 상세 판정 문구 확장 UI를 별도 검토한다.
- 모바일 하단 브라우저 UI와 기기별 safe area는 실제 기기에서 한 번 더 확인해야 한다.
