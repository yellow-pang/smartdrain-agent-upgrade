# 22 MVP 발표용 데이터 정확성 개선

## 변경 내용

- 수위 UI 필드를 `waterLevelCm`으로 바꾸고 수위 퍼센트·진행 바를 제거했다.
- 유량 표기를 유속(`m/s`)으로 통일했다. 막힘률만 퍼센트로 유지한다.
- 센서 이력은 `measuredAt` 오름차순으로 정렬하고, 현재 카드는 API `sensorSummary`를 우선 사용한다.
- 차트의 24시간·7일 탭, 하드코딩 기준선, 근거 없는 최고값 카드와 센서별 고정 상태 배지를 제거했다.
- 위험 점수 표기 대신 최종 상태와 최종 판단 문구를 반응형 카드 구조로 강조했다.
- API DTO의 누락 가능 값을 nullable로 반영하고, 화면에는 `-`로 표시한다. 상세 API 실패는 mock fallback 대신 오류 안내로 표시한다.

## 남은 확인

- 실제 FastAPI 응답으로 센서 이력이 최신순·오래된순 어느 방향으로 와도 차트와 현재값이 일관되게 보이는지 수동 확인이 필요하다.
- 발표 환경에서 API 장애 시 상세 오류 안내와 대시보드 오류 상태가 의도대로 표시되는지 확인이 필요하다.
