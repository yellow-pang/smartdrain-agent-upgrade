# 19 대시보드 현황 반응형 정보 밀도 개선 결과

## 변경 요약

대시보드 현황을 제거하지 않고, 관리자가 전체 규모를 먼저 이해한 뒤 위험 상태를 우선 확인하도록 상태 카드의 반응형 배치를 조정했다.

| 화면 크기 | 적용 배치 |
| --- | --- |
| 모바일 (`< 640px`) | `전체` 카드가 두 열 전체를 사용하고, 위험·주의·양호·판단불가는 두 열로 표시 |
| 태블릿 (`640px ~ 1023px`) | 첫 행에 `전체` 1열과 `위험` 2열, 다음 행에 주의·양호·판단불가 3열 표시 |
| 데스크톱 (`>= 1024px`) | 6단위 grid에서 `위험`만 2단위, 나머지 상태는 각 1단위로 표시 |

## 변경 파일

| 경로 | 변경 내용 |
| --- | --- |
| `app/page.tsx` | 실제 현황 카드와 로딩 스켈레톤에 같은 반응형 grid·span 규칙 적용 |
| `docs/plans/plan-17-dashboard-summary-responsive-layout.md` | 빈칸 제거보다 관제 정보 우선순위를 중심으로 계획 보완 |

## 변경 전/후

| 항목 | 변경 전 | 변경 후 |
| --- | --- | --- |
| 모바일 전체 현황 | 다른 상태 카드와 같은 두 열 중 한 칸 | 첫 행 전체를 사용해 관리 대상 규모를 먼저 표시 |
| 태블릿 위험 상태 | 다른 상태와 같은 한 칸 | 첫 행의 두 칸을 사용해 즉시 확인 대상을 강조 |
| 데스크톱 현황 영역 | 제목과 카드 묶음이 양끝으로 벌어짐 | 제목은 고정 폭, 상태 grid는 남은 폭을 채워 중앙 공백 축소 |
| 로딩 상태 | 실제 카드와 별도의 균등 grid | 실제 데이터와 같은 grid·span 규칙 |

## 남은 확인 사항

- 브라우저에서 360px, 640px, 768px, 1024px, 1280px 이상 폭을 확인해 계획한 카드 span과 텍스트 줄바꿈을 점검한다.
- 실제 위험 데이터가 여러 건일 때 넓어진 위험 카드가 상태 색상·레이블과 함께 충분히 눈에 띄는지 확인한다.
