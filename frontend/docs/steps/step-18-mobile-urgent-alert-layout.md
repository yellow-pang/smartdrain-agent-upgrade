# 18 모바일 긴급 알림 패널 레이아웃 수정 결과

## 작업 결과

- 모바일 화면에서 긴급 알림 패널을 알림 버튼 기준이 아닌 뷰포트 기준으로 배치했다.
- 기본 화면 크기에서는 패널에 좌우 `12px` 여백을 적용해 360px 폭에서도 왼쪽 텍스트가 잘리지 않게 했다.
- `sm` 이상 화면에서는 기존처럼 알림 버튼 아래 오른쪽에 표시되는 팝오버 동작을 유지했다.
- 알림 목록, 빈 상태, 읽음 처리, 시설 상세 링크의 기존 동작은 변경하지 않았다.

| `docs/plans/plan-16-mobile-urgent-alert-layout.md` | 문제 원인, 권장 방향, 검증 기준 기록 |

## 검증 결과

| 검증 | 결과 | 비고 |
| --- | --- | --- |
| `npm.cmd --prefix frontend run lint` | 통과 | 신규 error 없음. 기존 `components/fallback-image.tsx`의 native `<img>` warning 1건 유지 |
| `git diff --check` | 통과 | 공백 오류 없음 |
| 브라우저 수동 확인 | 필요 | 360×740, 390×844, 768px 이상에서 패널 위치와 텍스트 잘림을 확인해야 함 |

## 사용자 확인 방법

1. 브라우저 개발자 도구의 기기 모드에서 알림 아이콘을 눌러 긴급 알림 패널을 연다.
2. `360×740`과 `390×844`에서 제목, 설명, 빈 상태 문구의 왼쪽이 잘리지 않고 패널이 화면 안에 있는지 확인한다.
3. 긴급 알림이 여러 건인 상태에서 목록 스크롤과 `모두 읽음` 버튼이 정상 동작하는지 확인한다.
4. `768px` 이상에서 패널이 다시 알림 버튼 아래 오른쪽에 표시되는지 확인한다.
5. 알림 항목을 눌러 해당 시설 상세 화면으로 이동하고 패널이 닫히는지 확인한다.

## 남은 리스크

- 실제 긴 알림 문구와 다수 알림 데이터에서 패널 높이와 내부 스크롤의 체감 가독성을 추가 확인해야 한다.
- 모바일 기기의 주소 표시줄 등 동적 뷰포트 변화는 실제 기기에서도 한 번 확인하는 것이 좋다.
